# Repository Guidelines

A short guide for contributing to **codex-app**, an Android client (Kotlin +
Jetpack Compose) that bundles and runs a local Node.js (Hono) HTTP server via
Termux on the device.

## Project Structure & Module Organization

- `app/` — Android module. `app/src/main/java/com/codexapp/` holds Kotlin code
  organised by concern: `MainActivity.kt`, `ui/` (`CodexNavHost.kt`, `screens/`,
  `components/`, `theme/`), `network/` (`CodexClient.kt`, `ServerManager.kt`),
  and `server/CodexHttpServer.kt`.
- `app/src/main/res/` — Android resources (drawables, mipmaps, values/strings).
- `app/src/main/assets/server/` — bundled copy of the Node.js server sources
  (kept in sync with `server/`).
- `server/` — Standalone Hono server (`index.js`, `package.json`,
  `node_modules/`). `server.log` captures runtime output.
- `sessions/` — JSON files representing chat sessions keyed by UUID.
- Root Gradle files: `build.gradle.kts`, `settings.gradle.kts`,
  `app/build.gradle.kts`, `gradle.properties`, `gradlew[.bat]`.

## Build, Test, and Development Commands

Run from the repo root.

- `./gradlew assembleDebug` — build a debuggable APK
  (`app/build/outputs/apk/debug/app-debug.apk`).
- `./gradlew installDebug` — build and install onto a connected device.
- `./gradlew compileDebugKotlin` — fast Kotlin type-check only.
- `./gradlew clean` — remove `app/build/` outputs.
- `cd server && npm start` — run the bundled Hono server locally on the host
  for UI/HTTP testing without the device.
- `cd server && npm run dev` — same as `start` with `node --watch` for reload.

`MainActivity` boots `ServerManager` on launch; the server auto-runs
`npm install` if `server/node_modules/` is missing on the device.

## Wireless ADB Debugging

You can deploy and inspect the app without a USB cable using wireless ADB
over the loopback interface. The emulator/device is reachable on
`localhost:38979`.

- `adb connect localhost:38979` — pair the adb client with the device.
- `adb devices` — confirm `localhost:38979 device` is listed before running
  Gradle or `adb` commands.
- `./gradlew installDebug` then `adb shell am start -n com.codexapp/.MainActivity`
  to launch the freshly installed APK.
- `adb logcat -d -s ServerManager` — tail the in-app server lifecycle logs.
- `adb shell curl http://127.0.0.1:3000/api/health` — hit the bundled Hono
  server from the device shell.

If the connection drops (common after sleep), rerun `adb connect
localhost:38979`.

## Coding Style & Naming Conventions

- Kotlin: 4-space indent, `kotlin.code.style=official`. Classes in
  `PascalCase`, functions/properties in `camelCase`, constants in
  `UPPER_SNAKE_CASE`. Composable screens live in `ui/screens/`, reusable
  widgets in `ui/components/`.
- JavaScript/Node (`server/index.js`): 2-space indent, ES modules, double
  quotes, trailing commas where valid, `const`/`let` only.
- Keep bundled assets in sync: changes to `server/index.js` must be mirrored
  to `app/src/main/assets/server/`.

## Testing Guidelines

No automated test suite is currently configured (no `test/` or `androidTest/`
sources). Validate changes manually:

- `./gradlew compileDebugKotlin` for type safety.
- `npm start` in `server/` for endpoint smoke tests (`curl` against
  `/api/health`, `/api/sessions`, etc.).
- Install the debug APK and exercise chat/session flows end-to-end.

## Commit & Pull Request Guidelines

This checkout has no git history yet, so use these defaults until a convention
is adopted:

- Commits: imperative mood, ≤72-char subject, e.g.
  `server: handle missing sessions dir gracefully`.
- One logical change per commit; keep generated `app/build/` and
  `server/server.log` out of the index.
- Pull requests: short description, linked issue, steps to reproduce/verify,
  and screenshots or log excerpts for UI or server changes. Note any APK or
  bundled-asset updates.

## Agent-Specific Notes

- Do not edit files under `app/build/`; they are regenerated by Gradle.
- When changing the server, update both `server/` and
  `app/src/main/assets/server/` in the same patch.
- Preserve the `com.codexapp` namespace and `minSdk = 26` / `targetSdk = 34`
  unless the change requires otherwise.
- Prefer the wireless adb flow at `localhost:38979` over a USB cable for
  quick iteration.
